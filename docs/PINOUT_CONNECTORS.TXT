-----------------------------------------------------------------------------------------
1. CONN_MAIN (Power Input and CAN Bus)
   Type: B4B-XH-2 (2.54mm pitch)
-----------------------------------------------------------------------------------------
PIN | SIGNAL    | FUNCTION
-----------------------------------------------------------------------------------------
1   | +24V      | Main power supply (goes to the regulator and drivers)
2   | GND       | Common power ground
3   | CAN_H     | CAN Bus High (to TJA1057)
4   | CAN_L     | CAN Bus Low (to TJA1057)
    |
    | NOTE: whether this board carries a fixed 120-ohm termination
    |       resistor across CAN_H/CAN_L, or leaves it unpopulated/optional
    |       via a solder-bridge, isn't specified anywhere in this project's
    |       documentation yet. Doesn't matter if this board is always the
    |       last node on the bus - matters a lot if it's ever used as a
    |       middle node in a daisy chain, where a second fixed 120-ohm
    |       resistor would drag the bus impedance well below the 120-ohm
    |       (two terminators in parallel) the standard expects. Worth
    |       settling and documenting explicitly once the actual PCB
    |       decision is made.

4. CONN_DRILL (Multi-function: Drill / Laser / 4-Wire Fan)
   Type: B6B-XH-2 (2.54mm pitch)
------------------------------------------------------------------------------------------------
PIN | SIGNAL       | DRILL MODE           | 3D MODE (DELTA FAN)              | LASER MODE
------------------------------------------------------------------------------------------------
1   | ENABLE/BRAKE | PB9 (BL4260 brake,   | PB6 (Extruder ENN - NOT the fan) | PB6 (Safety interlock)
    |              |  own dedicated pin,  |                                   |
    |              |  see note below)     |                                   |
2   | FGIN         | PA3 (Tachometer)     | PA3 (Tachometer / BLUE wire)     | NC
3   | FRIN         | PA4 (Rotation IN/OUT)| NC                               | NC
4   | PWM          | PA8 (Speed control)  | PA8 (25kHz real PWM / YELLOW wire,| PA8 (Emission power)
    |              |                      |  TIM1_CH1, same channel as       |
    |              |                      |  drill/laser, reconfigured per-  |
    |              |                      |  tool - moved here from PA7 to   |
    |              |                      |  free that pin for the LED chain)|
5   | GND          | Ground               | Ground (BLACK wire)              | Ground
6   | +24V         | Power supply         | Power supply (RED wire)          | Power supply
    |              |                      |                                  |
    | CORRECTED: wire colors were previously swapped in this doc (had FG=Yellow,
    |   PWM=Blue). Cross-checked against the actual Delta EFB0424VHD-CP0
    |   datasheet: BLUE=FG, YELLOW=PWM. Re-check your harness if already built
    |   against the old documentation.
    | NOTE: the fan's "emergency stop" is a 1000ms CAN communication watchdog
    |       that zeroes the PWM autonomously - it does NOT touch PB6, since that
    |       line is already the extruder driver's enable in this same tool mode.
    | NOTE (NEEDS YOUR ATTENTION AT LAYOUT): Drill mode's ENABLE/BRAKE uses
    |       its own dedicated pin, PB9 - not shared with the stepper tools'
    |       TMC_ENN. This table still shows PB6 for
    |       3D/Laser mode on this same connector pin, matching the existing
    |       multi-mode pattern used throughout this document - but check
    |       whether that's actually what you want wired here: if CONN_DRILL's
    |       physical pin 1 ties BOTH PB6 and PB9 to the same net, make sure
    |       firmware never drives both as outputs at once for a mode where
    |       only one should be active (rather than assuming that's already
    |       handled safely). The simpler alternative is rewiring this pin to
    |       PB9 only, and moving the 3D-printer extruder-enable connection
    |       elsewhere (e.g. CONN_MOT, which already carries the real TMC2209
    |       driver lines for that tool) - whichever you decide, this is a
    |       trace-level call for your own Eagle work, not something assumed
    |       resolved by this document.

    |           |                       |                              | Soldering/Drill/Laser/AOI/Probe
------------------------------------------------------------------------------------------------
1   | GND       | Ground                | To ground (NTC Pin 1)        | Ground
2   | TCRT_A0   | PB0 (ADC_CH11)        | PB0 (ADC_CH11, NTC Pin 2 -   | NC
    |           |                       |  hardware 4.7k Pull-Up to    |
    |           |                       |  3.3V via R40 forms the      |
    |           |                       |  divider with the 100k NTC)  |
3   | TOUCH     | PB3 (Digital IN LM393)| NC                           | PB3 (Endstop/Probe Switch,
    |           |                       |                              |  active low, 330R R39 protected)
4   | +5V       | Sensor VCC supply     | NC                           | Probe VCC / NC

8. CONN_LED2 (Lighting - Camera/Inspection Coaxial Ring)
   Type: JST3-B3B-PH-K (2.00mm pitch)
-----------------------------------------------------------------------------------------
PIN | SIGNAL    | FIRMWARE LOGIC FUNCTION
-----------------------------------------------------------------------------------------
1   | GND       | Ground
2   | DATA_RING  | PB1 (Bit-banged, unchanged from earlier revisions -
    |            | independent from CONN_LED1's PA7/SPI1 above)
3   | +5V       | LED ring power
    |           | Feeds 2-3 addressable LED rings wired in PARALLEL on this
    |           | single data line (they mirror the same pattern - not
    |           | independently addressable from each other). Color and
    |           | on/off are set over CAN via 0x100 (rxData[4:6]=color,

9. CONN_EXPANSION (Add-on board header)
   Type: 20-pin, 1.27mm pitch, Würth eiCan series (grew from 14 pins - see BOM.TXT)
------------------------------------------------------------------------------------------------
PIN | SIGNAL         | FUNCTION
------------------------------------------------------------------------------------------------
1   | +24V           | Main rail passthrough
2   | GND            | Ground
3   | +5V            | Logic power
4   | EXP_SPI_MOSI   | PB15 - bit-banged (software) SPI, not a hardware SPI peripheral.
    |                | Was EXP_PWM in an earlier revision - see the note below for why it
    |                | moved here. See the SPI note below for why bit-banged.
5   | EXP_TMC_STEP   | PB12 - STEP pulses. Universal to any STEP/DIR driver on the
    |                | expansion board (TMC2209 and TMC5160 both use this the same way)
6   | EXP_TMC_EN     | PB14 - driver enable, universal same as EXP_TMC_STEP
7   | EXP_I2C2_SCL   | PB10 - own bus, separate from the OLED's I2C1. R48 (4.7k to
    |                | +3.3V) populated as an external pull-up
8   | EXP_SPI_CS     | PA2 - bit-banged, same bus as EXP_SPI_MOSI
9   | EXP_SPI_MISO   | PB8 - bit-banged, same bus as EXP_SPI_MOSI
10  | EXP_GPIO1      | PC14 - general purpose, EXTI-capable
11  | EXP_GPIO2      | PC15 - general purpose, EXTI-capable
12  | (no-connect)   | Not wired to an MCU pin. Was meant to carry EXP_SPI_MOSI via PD2
    |                | in an earlier revision of this design - PD2 does not exist on this
    |                | chip's LQFP48 package (confirmed against the official
    |                | STM32F303xB/xC datasheet's per-package pin table: the LQFP48 column
    |                | shows a dash for PD2, and in fact for every single Port D pin -
    |                | Port D is not broken out on this package at all). EXP_SPI_MOSI
    |                | moved to pin 4 (PB15) instead - see that entry's note.
13  | EXP_SPI_SCK    | PB2 - bit-banged, same bus as EXP_SPI_MOSI
14  | EXP_I2C2_SDA   | PB11. R44 (4.7k to +3.3V) populated as an external pull-up
15  | EXP_TMC_DIAG0  | NOT wired to an MCU pin (see note below) - this pin on the
    |                | connector is currently a no-connect
16  | EXP_TMC_DIR    | PB13 - direction, universal same as EXP_TMC_STEP
17  | GND            | Ground
18  | +3V3           | Logic power
19  | GND            | Ground
20  | +24V           | Main rail passthrough (second pin, for higher-current add-ons)
    |                |
    | NOTE - why EXP_SPI_MOSI is on PB15/pin 4, not PD2/pin 12 as in an earlier
    |       revision: PD2 was assumed available based on this document's own
    |       earlier "free pins" listing, which turned out to not have been
    |       checked against this specific package's actual pinout - a real
    |       error, not a deliberate design choice, caught on review. The
    |       official datasheet confirms Port D isn't present on LQFP48 at
    |       all. PB15 (previously EXP_PWM, not used by any current firmware
    |       logic) took its place - a reasonable trade since it was reserved,
    |       unused headroom either way, and it usefully doubles as the shared
    |       UART line for a TMC2209 populated instead of a TMC5160 (see the
    |       TMC2209/TMC5160 note below) - a role PD2 couldn't have filled
    |       even if it did exist, since it was never part of the shared
    |       STEP/DIR/EN group of pins.
    | NOTE - why bit-banged SPI, not a hardware peripheral: this chip's
    |       hardware SPI2 defaults to PB12/13/14/15 - exactly the pins
    |       already committed to EXP_TMC_STEP/DIR/EN and (now) EXP_SPI_MOSI
    |       above, so using it here would mean re-routing those first. SPI3's
    |       standard pins (PC10/PC11/PC12, confirmed against the same
    |       official datasheet's LQFP48-specific pin table) aren't present
    |       on this package at all either - only on the larger LQFP64/LQFP100
    |       variants of the same silicon. Bit-banged SPI on 4 of the board's
    |       remaining free pins avoids re-routing anything already committed,
    |       and this firmware already has a working precedent for bit-banged
    |       serial timing (the WS2812B ring driver on PB1) - SPI's per-bit
    |       timing requirements are far more forgiving than WS2812B's, so
    |       this is a comfortably proven approach for this codebase, not a
    |       novel one. TMC5160 configuration doesn't need anywhere near a
    |       hardware peripheral's top speed to work correctly. Confirmed
    |       against the TMC5160's own datasheet: it specifies SPI Mode 3
    |       (CPOL=1, CPHA=1) explicitly by name, which the firmware's
    |       bit-banged implementation matches exactly - see
    |       ExpansionSPI_TransferByte() in STM32F303CC.C.
    | NOTE - why EXP_TMC_DIAG0 isn't wired: with the pin budget corrected
    |       above, this board still has exactly 4 MCU pins available with no
    |       other commitment (PA2, PB2, PB8, PB15), all four already spoken
    |       for by the SPI bus - one short of the 5th needed for DIAG0. DIAG0
    |       is a hardware interrupt line for immediate stall/fault
    |       notification; without it, the same information (TMC5160's
    |       DRV_STATUS register - stall flag, overtemperature warning,
    |       open-load) is still readable over the SPI bus above, just via
    |       periodic polling instead of an instant interrupt. Wiring a
    |       dedicated DIAG0 line would need sacrificing something else
    |       already assigned here - worth a deliberate decision, not a side
    |       effect of this change.
    | NOTE: an I2C ADC/DAC chip or a port-expander chip on the add-on board
    |       both hang off EXP_I2C2_SCL/EXP_I2C2_SDA like any other I2C device
    |       - no dedicated pins needed for those specifically. See
    |       PINOUT.TXT Block 10 and the README's "Expansion Connector"
    |       section.
    | NOTE - TMC2209 and TMC5160 sharing: the expansion board is expected to
    |       carry one or the other, not necessarily both - a TMC2209 uses
    |       only EXP_TMC_STEP/DIR/EN plus its own single-wire UART
    |       (TMCUART, half-duplex), never the 4 EXP_SPI_* pins, which have
    |       no meaning to that chip at all. Since the two populate options
    |       are mutually exclusive on any given board, EXP_SPI_MOSI (PB15,
    |       pin 4) doubles as a natural home for a TMC2209's single UART
    |       line when that's the chip actually populated - no separate
    |       dedicated UART pin was added, since one is never needed at the
    |       same time as the SPI bus is. If a specific board design ever
    |       does want both chips populated together, EXP_TMC_STEP/DIR/EN can
    |       still be wired to both drivers in parallel like any shared
    |       STEP/DIR bus, but the TMC2209 would then need its own separate
    |       UART pin rather than sharing pin 4 with the TMC5160's MOSI -
    |       that configuration isn't part of this revision.

========================================================================================
FREE MCU PINS (not on any connector): none of PA2, PB2, PB8, PB15 remain free
as of this revision - all four are now committed to the bit-banged
EXP_SPI_* bus above. PB1 is NOT free - still drives CONN_LED2, unchanged.
PA8 is NOT free either - see the drill/laser PWM table above. PC13 carries
ID4 (not free), PB9 carries the drill's dedicated brake (not free),
PB10-PB14/PC14-PC15 feed CONN_EXPANSION above (not free). No MCU pins
remain unallocated on this design as of this revision. Port D is not
present at all on this chip's LQFP48 package (confirmed against the
official STM32F303xB/xC datasheet's per-package pin table) - not a source
of any free pins on this board, ever, regardless of what's used elsewhere.
